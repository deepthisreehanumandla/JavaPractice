# 📘 Video 24 - JDK, JRE & JVM

## 1. Concepts

### What is JDK?

**JDK (Java Development Kit)** is used to **develop Java applications**.

It contains:

* JRE
* JVM
* Java Compiler (`javac`)
* Development tools

Without the JDK, you **cannot compile** Java programs.

---

### What is JRE?

**JRE (Java Runtime Environment)** is used to **run Java programs**.

It contains:

* JVM
* Required libraries

The JRE provides the environment needed to execute Java applications.

---

### What is JVM?

**JVM (Java Virtual Machine)** is responsible for **executing Java bytecode**.

It converts bytecode into machine code so that the operating system can understand and execute it.

---

## 2. Why?

Java follows the principle:

> **Write Once, Run Anywhere (WORA).**

The same bytecode can run on different operating systems because each operating system has its own JVM.

---

## 3. Program Execution Flow

```text
Java Source Code (.java)
          │
          ▼
     javac Compiler
          │
          ▼
   Bytecode (.class)
          │
          ▼
         JVM
          │
          ▼
     Machine Code
          │
          ▼
       Program Runs
```

---

## 4. Key Differences

### JDK vs JRE vs JVM

| JDK                           | JRE                       | JVM                               |
| ----------------------------- | ------------------------- | --------------------------------- |
| Development Kit               | Runtime Environment       | Virtual Machine                   |
| Used to develop Java programs | Used to run Java programs | Executes bytecode                 |
| Contains JRE                  | Contains JVM              | Converts bytecode to machine code |

---

### Compiler vs JVM

| Compiler (`javac`)          | JVM                              |
| --------------------------- | -------------------------------- |
| Converts `.java` → `.class` | Converts bytecode → Machine Code |
| Part of JDK                 | Part of JRE                      |

## 5. Interview Questions

### Q1. What is JDK?

**Answer:**

JDK (Java Development Kit) is used to develop Java applications. It contains the JRE, JVM, compiler, and development tools.

---

### Q2. What is JRE?

**Answer:**

JRE (Java Runtime Environment) provides the environment required to run Java applications.

---

### Q3. What is JVM?

**Answer:**

JVM (Java Virtual Machine) executes Java bytecode by converting it into machine code.

---

### Q4. Does JDK contain JRE?

**Answer:**

Yes.

---

### Q5. Does JRE contain JVM?

**Answer:**

Yes.

---

### Q6. Which component compiles Java source code?

**Answer:**

`javac` compiler.

## 7. Key Takeaways

* JDK is used to develop Java applications.
* JRE is used to run Java applications.
* JVM executes Java bytecode.
* `javac` converts `.java` into `.class`.
* JVM converts bytecode into machine code.
* JDK ⊃ JRE ⊃ JVM.

---

## 8. Common Mistakes

❌ Thinking JDK, JRE, and JVM are the same.

✔️ They have different roles.

---

❌ Thinking JVM compiles Java source code.

✔️ The **compiler (`javac`)** compiles `.java` files. The **JVM** executes the generated bytecode.

---

❌ Thinking JRE is enough to write Java programs.

✔️ You need the **JDK** to develop and compile Java programs.

---

## 📝 30-Second Revision

* **JDK** → Develop Java programs.
* **JRE** → Run Java programs.
* **JVM** → Execute bytecode.
* `javac` → `.java` → `.class`
* JVM → Bytecode → Machine Code.
* **JDK contains JRE, and JRE contains JVM.**
